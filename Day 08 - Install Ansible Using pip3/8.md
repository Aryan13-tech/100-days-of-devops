# 🚀 Day 08 - Install Ansible Using pip3

## 📌 Challenge Overview

During the weekly meeting, the **Nautilus DevOps team** decided to start using **Ansible** for automation and configuration management due to its simplicity and minimal prerequisites.

To begin testing, they selected the **Jump Host** as the **Ansible controller** to execute automation tasks on the rest of the servers in the **Stratos Datacenter**.

The task was to install **Ansible version 4.9.0** on the **Jump Host** using **pip3** and ensure the Ansible command is available globally for all users.

---

# 🎯 Objective

Install **Ansible 4.9.0** on the **Jump Host** using **pip3** and make sure the `ansible` command is accessible globally.

---

# 🖥 Infrastructure Details

| Server    | Hostname  | User |
| --------- | --------- | ---- |
| Jump Host | jump_host | thor |

---

# 🔑 Step 1 — Login to Jump Host

Access the jump host using the **thor** user.

```bash
ssh thor@jump_host
```

---

# 🔑 Step 2 — Switch to Root User

```bash
sudo -i
```

---

# 📦 Step 3 — Install Python pip3

Install the pip3 package manager.

```bash
yum install -y python3-pip
```

---

# 📦 Step 4 — Install Ansible 4.9.0 Using pip3

```bash
pip3 install ansible==4.9.0
```

This installs:

* ansible
* ansible-core
* required Python dependencies

---

# 🌍 Step 5 — Make Ansible Available Globally

Create a symbolic link so that all users can run Ansible commands.

```bash
ln -s /usr/local/bin/ansible /usr/bin/ansible
```

---

# 🔍 Step 6 — Verify Installation

Check if Ansible is installed correctly.

```bash
ansible --version
```

Example output:

```
ansible [core 2.11.x]
python version = 3.x
```

---
