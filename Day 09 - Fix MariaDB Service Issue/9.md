# 🚀 Day 09 - Fix MariaDB Service Issue

## 📌 Challenge Overview

The **Nautilus application** in the **Stratos Datacenter** was facing a critical issue.
The production support team discovered that the application could not connect to the database.

After investigation, it was found that the **MariaDB service on the Database Server was down**.

The task was to **troubleshoot the issue and bring the MariaDB service back online** so the application could reconnect to the database.

---

# 🎯 Objective

Fix the issue on the **Database Server (stdb01)** and ensure the **MariaDB service is running properly**.

---

# 🖥 Infrastructure Details

| Server          | Hostname | User  |
| --------------- | -------- | ----- |
| Database Server | stdb01   | peter |

---

# 🔑 Step 1 — Connect to Database Server

Login from the jump host.

```bash
ssh peter@stdb01
```

---

# 🔑 Step 2 — Switch to Root User

```bash
sudo -i
```

---

# 🔧 Step 3 — Stop MariaDB Service

```bash
systemctl stop mariadb
```

---

# 🧹 Step 4 — Reset Database Directory

Remove corrupted database files.

```bash
rm -rf /var/lib/mysql/*
```

---

# 🗄 Step 5 — Initialize MariaDB Database

```bash
mariadb-install-db --user=mysql
```

This command creates the initial MariaDB system database structure.

---

# 🔐 Step 6 — Fix Permissions

```bash
chown -R mysql:mysql /var/lib/mysql
```

---

# ▶ Step 7 — Start MariaDB Service

```bash
systemctl start mariadb
```

---

# 🔍 Step 8 — Verify Service Status

```bash
systemctl status mariadb
```

Expected output:

```
Active: active (running)
```

---

# ⚙ Key Concepts Learned

* Troubleshooting service failures
* Managing services using `systemctl`
* Resetting database directories
* Initializing MariaDB databases
* Fixing permission issues for services
