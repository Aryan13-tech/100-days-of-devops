# 🚀 Day 02 - Temporary User Setup with Expiry Date

## 📌 Challenge Overview

As part of the **100 Days of DevOps Challenge (KodeKloud Engineer)**,  
the task was to create a temporary user account for a developer named **siva** with a defined expiry date.

This ensures controlled and time-bound access to the system.

---

## 🖥 Infrastructure Details

- Jump Host: `jumphost`
- Target Server: `stapp03` (App Server 3)
- Login User: `banner`
- Privilege Escalation: `sudo` to root

---

## 🎯 Objective

Create a Linux user with:

- Username: `siva`
- Expiry Date: `2027-01-28`
- Ensure username is lowercase
- Configure on App Server 3

---

# 🛠 Step-by-Step Implementation

## 1️⃣ Connect to App Server 3 from Jump Host

```bash
ssh banner@stapp03
```

---

## 2️⃣ Switch to Root User

```bash
sudo -i
```

---

## 3️⃣ Create User with Expiry Date

```bash
useradd siva -e 2027-01-28
```

Explanation:

- `useradd` → creates new user
- `-e` → sets account expiry date
- `2027-01-28` → expiry date in YYYY-MM-DD format

---

## 4️⃣ Verify Expiry Configuration

```bash
chage -l siva
```

Expected Output:

```
Account expires : Jan 28, 2027
```

---

# ✅ Final Result

✔ User `siva` successfully created  
✔ Expiry date set correctly  
✔ Configured on correct server (stapp03)  
✔ Verified using `chage -l`  
✔ Task passed successfully  

---

# 🔐 Why Set Expiry Dates?

Temporary accounts are useful for:

- Contract developers
- Short-term project access
- Vendor access
- Security compliance requirements

Setting an expiry date ensures automatic access removal without manual intervention.

---

# 🧠 Key Learnings

- Creating users with expiration using `useradd -e`
- Checking account status using `chage -l`
- Managing temporary access securely
- Applying real-world DevOps access control practices
