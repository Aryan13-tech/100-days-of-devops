# 🚀 Day 03 - Disable Direct Root SSH Login

## 📌 Challenge Overview

As part of the **100 Days of DevOps Challenge (KodeKloud Engineer)**,  
the task was to disable direct root SSH login on all application servers in the **Stratos Datacenter**.

This was implemented as part of a new security protocol rollout.

---

## 🖥 Infrastructure Details

Affected Servers:

- `stapp01`
- `stapp02`
- `stapp03`

Access Method:

- Jump Host: `jumphost`
- Privilege escalation using `sudo`

---

## 🎯 Objective

Disable direct SSH login for the root user on all app servers by modifying the SSH daemon configuration.

---

# 🛠 Step-by-Step Implementation

The following steps were performed on **each app server**.

---

## 1️⃣ Connect to Server

Example (for stapp01):

```bash
ssh tony@stapp01
```

(Similarly used `steve` for stapp02 and `banner` for stapp03)

---

## 2️⃣ Switch to Root User

```bash
sudo -i
```

---

## 3️⃣ Remove Any Existing Root Login Entries

```bash
sed -i '/PermitRootLogin/d' /etc/ssh/sshd_config
```

This ensures:
- No duplicate entries
- No conflicting configurations

---

## 4️⃣ Explicitly Disable Root Login

```bash
echo "PermitRootLogin no" >> /etc/ssh/sshd_config
```

This guarantees the configuration is correctly set in the main SSH configuration file.

---

## 5️⃣ Restart SSH Service

```bash
systemctl restart sshd
```

---

## 6️⃣ Verification

```bash
grep PermitRootLogin /etc/ssh/sshd_config
```

Expected Output:

```
PermitRootLogin no
```

This step was verified on:

- stapp01
- stapp02
- stapp03

---

# ✅ Final Result

✔ Direct root SSH login disabled  
✔ SSH configuration updated correctly  
✔ Service restarted successfully  
✔ Verified on all app servers  
✔ Task passed successfully

# 🔐 Why Disable Root SSH Login?

Disabling direct root SSH login is a critical security best practice because:

- Prevents brute-force attacks on root
- Enforces least-privilege access
- Encourages proper sudo usage
- Improves auditability and accountability

This is a standard security control in production environments.
