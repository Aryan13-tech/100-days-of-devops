# 🚀 Day 04 - Script Execution Permissions

## 📌 Challenge Overview

As part of the **100 Days of DevOps Challenge (KodeKloud Engineer)**,  
the sysadmin team at **xFusionCorp Industries** developed a backup automation script named:

```
xfusioncorp.sh
```

However, the script was missing executable permissions on **App Server 2** in the **Stratos Datacenter**.

The task was to grant proper permissions so **all users can execute the script**.

---

# 🖥 Infrastructure Details

| Server | Hostname | User |
|------|------|------|
| App Server 2 | stapp02 | steve |

Script Location:

```
/tmp/xfusioncorp.sh
```

---

# 🎯 Objective

Grant executable permissions to the script so **all users can execute it**.

---

# 🛠 Step-by-Step Implementation

## 1️⃣ Connect to App Server 2

```bash
ssh steve@stapp02
```

---

## 2️⃣ Switch to Root User

```bash
sudo -i
```

---

## 3️⃣ Grant Execute Permission

Assign proper permissions to the script.

```bash
chmod 755 /tmp/xfusioncorp.sh
```

Permission breakdown:

| Permission | Meaning |
|---|---|
| 7 | Read + Write + Execute (Owner) |
| 5 | Read + Execute (Group) |
| 5 | Read + Execute (Others) |

---

## 4️⃣ Verify Permissions

```bash
ls -l /tmp/xfusioncorp.sh
```

Expected output:

```
-rwxr-xr-x 1 root root /tmp/xfusioncorp.sh
```

This confirms that **all users can execute the script**.

---

# ✅ Final Result

✔ Executable permission granted  
✔ Script accessible to all users  
✔ Permissions verified successfully  
✔ Task passed successfully  

---

# 🧠 Key Learnings

- Understanding Linux file permissions
- Using the `chmod` command
- Numeric permission notation (755)

- Managing script execution rights in Linux
