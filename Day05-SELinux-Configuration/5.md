# 🚀 Day 05 - SELinux Installation and Configuration

## 📌 Challenge Overview

Following a security audit, the **xFusionCorp Industries** security team decided to enhance system security using **SELinux**.

As part of the testing phase, the task was assigned on **App Server 3** in the **Stratos Datacenter**.

---

# 🎯 Objectives

The following tasks were required:

1. Install the required **SELinux packages**.
2. Permanently **disable SELinux** (temporarily for testing).
3. No server reboot is required.
4. The final status **after reboot should be `disabled`**.

---

# 🖥 Infrastructure Details

| Server       | Hostname | User   |
| ------------ | -------- | ------ |
| App Server 3 | stapp03  | banner |

---

# 🔐 Step 1 — Connect to the Server

```bash
ssh banner@stapp03
```

---

# 🔐 Step 2 — Switch to Root User

```bash
sudo -i
```

---

# 📦 Step 3 — Install Required SELinux Packages

```bash
yum install -y selinux-policy selinux-policy-targeted policycoreutils
```

These packages provide:

* SELinux policy rules
* Targeted SELinux policies
* Management utilities

---

# ⚙ Step 4 — Disable SELinux Permanently

Edit the SELinux configuration file:

```bash
vi /etc/selinux/config
```

Find the following line:

```bash
SELINUX=enforcing
```

Change it to:

```bash
SELINUX=disabled
```

Save and exit the file.

---

# 🔍 Step 5 — Verify Configuration

```bash
cat /etc/selinux/config
```

Expected output:

```
SELINUX=disabled
```

---

# ⚠ Important Note

The task specifically mentions:

> Ignore the current status of SELinux.
> After the scheduled reboot, the status should be **disabled**.

Therefore **no reboot is required** for this task.
