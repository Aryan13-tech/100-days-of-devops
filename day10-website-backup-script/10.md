# Day 10 – Website Backup Automation Script

## Task

The production support team at xFusionCorp Industries required a **Bash script** to automate website backups.

The script must:

1. Create a zip archive of the website directory.
2. Save the archive in the local `/backup` directory.
3. Copy the archive to the Nautilus Storage Server.
4. Perform the copy **without asking for a password**.
5. Be stored in `/scripts` directory.
6. Avoid using `sudo` inside the script.

---

# Infrastructure Details

| Server               | Hostname   | User      |
| -------------------- | ---------- | --------- |
| Application Server 3 | `stapp03`  | `banner`  |
| Storage Server       | `ststor01` | `natasha` |

Website location:

```
/var/www/html/official
```

Temporary backup location:

```
/backup
```

Script location:

```
/scripts
```

---

# Install Required Package

The `zip` package must be installed manually before running the script.

```bash
sudo yum install zip -y
```

---

# Backup Script

File:

```
/scripts/official_backup.sh
```

Script content:

```bash
#!/bin/bash

zip -r /backup/xfusioncorp_official.zip /var/www/html/official
scp /backup/xfusioncorp_official.zip natasha@ststor01:/backup/
```

Make it executable:

```bash
chmod +x /scripts/official_backup.sh
```

---

# Passwordless SSH Setup

Generate SSH key:

```bash
ssh-keygen
```

Copy the key to the storage server:

```bash
ssh-copy-id natasha@ststor01
```

This ensures the script runs without asking for a password.

---

# Run the Script

```bash
/scripts/official_backup.sh
```

---

# Result

✔ Website directory archived
✔ Backup saved in `/backup`
✔ Archive copied to Storage Server
✔ Script runs without password

Backup file:

```
xfusioncorp_official.zip
```
