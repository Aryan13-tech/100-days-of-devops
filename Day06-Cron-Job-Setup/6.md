# 🚀 Day 06 - Cron Job Setup on Multiple Servers

## 📌 Challenge Overview

The **Nautilus system administrators** prepared scripts to automate routine operational tasks across the infrastructure. Before deploying them in production, they wanted to test scheduling functionality using a **sample cron job**.

The task was performed on **all Nautilus App Servers in the Stratos Datacenter**.

---

# 🎯 Objectives

1. Install the **cronie** package on all Nautilus app servers.
2. Start and enable the **crond service**.
3. Add a **cron job for the root user** that runs every 5 minutes.

Cron Job Requirement:

```
*/5 * * * * echo hello > /tmp/cron_text
```

---

# 🖥 Infrastructure Details

| Server       | Hostname | User   |
| ------------ | -------- | ------ |
| App Server 1 | stapp01  | tony   |
| App Server 2 | stapp02  | steve  |
| App Server 3 | stapp03  | banner |

---

# 🔐 Step 1 — Connect to App Server 1

```bash
ssh tony@stapp01
```

Switch to root:

```bash
sudo -i
```

Install cronie:

```bash
yum install -y cronie
```

Start and enable cron service:

```bash
systemctl start crond
systemctl enable crond
```

Add the cron job:

```bash
(crontab -l 2>/dev/null; echo "*/5 * * * * echo hello > /tmp/cron_text") | crontab -
```

Exit server:

```bash
exit
exit
```

---

# ⚙ What This Cron Job Does

| Field | Meaning              |
| ----- | -------------------- |
| */5   | Runs every 5 minutes |
| *     | Every hour           |
| *     | Every day            |
| *     | Every month          |
| *     | Every weekday        |

Command executed:

```
echo hello > /tmp/cron_text
```

This writes **"hello"** to the file `/tmp/cron_text`.
